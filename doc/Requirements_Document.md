## 1. 项目背景与目标

### 1.1 背景

自学者通常会购买或收集大量本地视频课程（编程、设计、语言等）。现有问题：
- 课程文件分散在硬盘各处，缺乏统一管理入口
- 学习进度难以追踪：看到哪了？哪些章节还没看？
- 缺乏学习规划：没有明确的学习计划和进度反馈
- 视频播放器与学习管理分离，切换成本高

### 1.2 核心目标

| 目标 | 描述 |
|------|------|
| **统一管理** | 一键导入本地课程文件夹，自动扫描视频文件并建立课程库 |
| **进度追踪** | 精确到每个视频的观看进度，支持断点续播 |
| **学习规划** | 支持制定周学习计划，追踪实际投入与计划的偏差 |
| **数据可视化** | 以热力图、统计面板等形式直观展示学习数据 |
| **沉浸体验** | 内置视频播放器，学习与管理不分离，支持变速播放 |

## 2. 用户画像与使用场景

### 2.1 目标用户

- **自学程序员**：购买了多套编程课程，需要系统化管理学习进度
- **备考学生**：需要按计划推进视频课程，追踪每日投入
- **终身学习者**：有多门并行课程，需要清晰的学习概览

### 2.2 核心使用场景

| 场景 | 描述 | 优先级 |
|------|------|--------|
| 课程导入 | 用户选择本地文件夹，系统自动扫描视频文件并创建课程条目 | P0 |
| 日常学习 | 用户打开课程，选择视频播放，系统自动记录观看进度 | P0 |
| 进度概览 | 用户在首页查看所有课程的学习进度、今日投入、学习余额 | P0 |
| 计划制定 | 用户为课程设置每周每天的学习时长目标 | P1 |
| 数据分析 | 用户查看学习热力图、完成率、预计完成日期等统计 | P1 |
| 断点续播 | 用户重新打开课程时，从上次观看位置继续 | P1 |
| 变速播放 | 用户以 0.5x ~ 2.0x 速度播放视频 | P2 |

## 3. 功能需求

### 3.1 课程库管理（HomeView）

#### FR-3.1.1 课程导入
- 用户点击「添加课程」按钮，系统弹出文件夹选择对话框
- 系统递归扫描选中文件夹，识别所有视频文件（.mp4 / .mkv / .avi / .mov / .flv / .wmv / .webm / .m4v）
- 扫描过程显示进度对话框（异步执行，不阻塞 UI）
- 扫描完成后计算：视频总数、总时长
- 课程名称默认使用文件夹名，支持后续修改
- **验收标准**：1000 个视频文件的文件夹扫描完成时间 < 10 秒

#### FR-3.1.2 首页仪表盘
- 在课程网格上方显示一行汇总统计卡片（4 张等宽卡片），一目了然地展示全局学习状态：
  1. **📚 课程总览**：总课程数 + 进行中数量（如 "3 门课程 / 2 门进行中"）
  2. **✅ 整体进度**：所有课程聚合的视频完成率（如 "已完成 15/48 个视频"）+ 进度条
  3. **⏱ 今日学习**：今日所有课程累计学习时长（如 "1h30m"）+ 与计划对比的百分比
  4. **🔥 连续学习**：连续有学习记录的天数（如 "连续 7 天"）
- 每张卡片为圆角矩形（12px），统一样式，内边距一致
- 卡片数据跟随课程增删、学习进度实时更新
- 无课程时显示占位状态（数字为 0 / "--"）
- **验收标准**：仪表盘刷新与课程列表同步，数据准确反映 data_manager 中的数据

#### FR-3.1.3 课程列表展示
- 以卡片网格布局展示所有课程（默认 3 列，窗口缩小时自动调整列数）
- 每张课程卡片显示：
  - 课程名称（支持双击编辑）
  - 整体进度百分比 + 进度条（带动画过渡）
  - 课程总时长（已观看 / 总时长，格式 HH:MM）
  - 今日学习时长（实际 / 计划，格式 HH:MM）
  - 预计剩余天数
  - 学习余额（正值绿色表示超前，负值红色表示落后）
- 卡片支持 hover 高亮效果
- **验收标准**：卡片数据在刷新后 200ms 内完成渲染

#### FR-3.1.4 课程删除
- 右键课程卡片弹出上下文菜单，包含「删除课程」选项
- 点击删除后弹出确认对话框
- 确认后删除课程及其所有进度数据
- **验收标准**：删除操作需二次确认，防止误操作

#### FR-3.1.5 课程去重
- 导入时检查课程路径是否已存在，若重复则提示用户
- **验收标准**：同一路径不可重复添加

### 3.2 视频播放（DetailView - Player）

#### FR-3.2.1 播放引擎
- **主引擎**：VLC (libvlc)，支持硬件解码，提供 `--audio-time-stretch` 变速不变调
- **备用引擎**：Qt Multimedia (QMediaPlayer)，当 VLC 不可用时自动降级
- 系统启动时自动检测 VLC 可用性
- **验收标准**：VLC 检测失败时，自动切换至 Qt 引擎并给出提示

#### FR-3.2.2 播放控制
- 播放/暂停（支持空格键快捷键）
- 进度条拖拽跳转
- 音量调节（0-100%）
- 播放速度调节（0.5x / 0.75x / 1.0x / 1.25x / 1.5x / 2.0x）
- **验收标准**：所有控制操作响应延迟 < 100ms

#### FR-3.2.3 播放器 UI
- 悬浮式控制栏（在视频区域上方浮动，鼠标移动时显示，静止 3 秒后自动隐藏）
- 控制栏跟随窗口大小、移动同步缩放和定位
- 切换应用窗口时自动隐藏控制栏（防止悬浮在其他应用上方）
- 最大化/窗口化时控制栏圆角自适应

#### FR-3.2.4 断点续播
- 每次打开视频，从上次观看位置继续播放（精度：秒级）
- 视频完成后不自动续播（等待用户手动选择下一个视频）
- **验收标准**：进度记录在视频切换时立即保存

#### FR-3.2.5 沉浸模式
- 点击「沉浸学习」按钮或全屏按钮，隐藏侧边栏和顶部导航
- 视频区域扩展至全窗口，左下角出现圆角
- 再次点击恢复
- **验收标准**：沉浸模式切换动画流畅，视频区域自适应

#### FR-3.2.6 侧边栏
- 按文件夹结构组织视频列表（章节折叠/展开）
- 每个视频项显示：状态图标（未看📄 / 观看中🕒 / 已完成✅）、文件名、时长
- 当前播放视频高亮显示（带主题色左边框）
- 点击任意视频切换播放
- **验收标准**：侧边栏支持滚动，500+ 视频不卡顿

### 3.3 进度追踪

#### FR-3.3.1 学习进度记录
- 每秒更新当前视频的观看时长
- 观看超过视频总时长 90% 时标记为「已完成」
- 完成的视频在列表中显示 ✅ 图标
- **验收标准**：进度数据在播放过程中每 5 秒自动持久化

#### FR-3.3.2 每日学习统计
- 按日期记录每日学习时长（秒）
- 同一视频多次观看，仅增量计入当日统计
- **验收标准**：跨天学习正确分天统计

#### FR-3.3.3 活动日志
- 每日完成的视频数量记录
- 用于激励反馈（如连续学习天数）

### 3.4 学习计划与分析（PropertiesView / 课程看板）

#### FR-3.4.1 学习计划卡片（只读展示 + 弹窗编辑）

学习计划卡片采用「展示/编辑分离」设计：

**A. 只读展示卡片（StudyPlanCard）**

卡片以只读模式展示当前学习计划的核心信息，不提供任何直接编辑控件：

- **标题栏**：左侧 "📋 学习计划" 标题，右侧 "✏️ 更改学习计划" 按钮。
- **开始日期**：以文本形式显示（如 "📅 开始日期：2026年6月19日"），不可直接修改。
- **每日时长可视化**：7 个横向迷你进度条（周一至周日），每个进度条上方显示星期名、下方显示小时数（如 "1.5h"）。进度条范围 0-12h，按比例填充主题色。
- **每周合计**：显示 "📊 每周合计：X.X 小时"，自动汇总每日时长。
- **结果卡片**：
  - 🏁 预计完成：预计完成日期 + 剩余天数
  - ⚖ 学习余额：超前/落后小时数（绿色/红色）
- **学习建议**：底部自动显示人性化建议文字（同 v1.0 逻辑）。

**B. 编辑弹窗（PlanEditDialog）**

点击「✏️ 更改学习计划」按钮弹出模态对话框，包含所有编辑控件：

- **开始日期**：内嵌 `ElaDatePicker`，初始值为当前设置的日期。
- **双页签切换**：
  - 页签 1「🎯 整体调节」：
    - 预设方案按钮组（5 个）：清空 / 0.5h/天 / 1h/天 / 2h/天 / 3h/天，点击后统一设置每日时长
    - 工作日组滑块：拖动后批量更新周一至周五
    - 周末组滑块：拖动后批量更新周六至周日
  - 页签 2「📋 每日微调」：
    - 7 个横向滑块（周一至周日），每滑块范围 0-12h，步长 10 分钟
    - 滑块值实时显示在右侧标签
- **底部按钮**：「取消」（关闭弹窗，数据不变）和「确认更改」（保存并更新）。
- **双向同步**：两个页签的滑块数据实时同步——在页签 1 拖动工作日滑块，页签 2 的周一至周五滑块同步更新，反之亦然。

**C. 交互流程**

1. 用户查看只读卡片，了解当前学习计划状态。
2. 点击「✏️ 更改学习计划」按钮 → PlanEditDialog 弹出。
3. 用户在弹窗中修改日期、使用预设或调整滑块。
4. 点击「确认更改」→ 弹窗关闭 → `plan_changed` 信号发射 → Controller 持久化 → 看板刷新 → 卡片展示新数据。
5. 点击「取消」→ 弹窗关闭，数据不变。

- **验收标准**：弹窗确认后，卡片数据、结果卡片、学习建议和时间线在 500ms 内同步更新。

#### FR-3.4.2 学习余额计算
- 余额 = 实际累计学习时长 - 计划累计学习时长（从开始日期至今天）
- 正值 = 超前于计划（绿色），负值 = 落后于计划（红色）
- 余额在统计卡片和时间线卡片中同步展示

#### FR-3.4.3 统计卡片面板
以 4 张等宽卡片展示课程核心数据：
- 📚 **视频进度**：已完成数 / 总数 + 百分比进度条
- ⏱ **累计时长**：已观看 / 总时长（小时）+ 完成百分比
- 📅 **今日概览**：今日学习 / 今日计划（小时）+ 完成百分比
- ⚖ **学习余额**：超前/落后小时数（绿色/红色）

#### FR-3.4.4 学习时间线（纯可视化）

- 以可视化进度条展示从「开始日期 → 今天 → 预计完成」的学习旅程
- 日期编辑功能已移至「学习计划卡片」（FR-3.4.1），时间线卡片仅用于可视化展示
- 时间线包含以下元素：
  - 🚩 开始日期（在进度条左侧标注，不可点击）
  - 📍 今天标记（在进度条上标注当前位置）
  - 🏁 预计完成日期（在标题栏右侧显示）
  - 进度条分段着色：已完成部分（主题色）、未完成部分（浅灰）
  - 底部标签：已过天数 / 预计总天数
- 如果未设置开始日期或周计划，显示引导提示
- 如果课程已全部完成，显示「🎉 课程已完成」
- **验收标准**：修改学习计划卡片中的日期或周计划后，时间线立即更新

### 3.5 全局功能

#### FR-3.5.1 主题切换
- 支持浅色（Light）和深色（Dark）两种主题
- 切换时以点击位置为中心播放圆形扩散动画
- **验收标准**：主题切换动画时长 ~700ms，60fps 流畅

#### FR-3.5.2 自定义窗口
- 无边框窗口，自定义标题栏
- 窗口控件：最小化、最大化/还原、关闭
- 窗口阴影（非最大化状态）
- 窗口圆角（12px，最大化时取消）
- 标题栏拖拽移动
- 双击标题栏切换最大化/还原
- 边缘拖拽调整窗口大小（8px 可交互区域）
- **验收标准**：窗口行为与原生 Windows 窗口一致

#### FR-3.5.3 数据持久化
- 所有数据存储为本地 JSON 文件（`data/courses.json`）
- UTF-8 编码，`ensure_ascii=False` 保证中文可读
- 应用启动时自动加载，操作后自动保存
- **验收标准**：数据文件手动编辑后，应用启动不崩溃，显示默认空状态

## 4. 数据模型

### 4.1 核心实体

```
Course（课程）
├── id: UUID string                    # 唯一标识
├── name: string                       # 课程名称
├── path: string                       # 课程文件夹绝对路径
├── added_at: ISO datetime             # 添加时间
├── total_videos: int                  # 视频总数
├── total_duration: float              # 总时长（秒）
├── start_date: ISO date | null        # 计划开始日期
├── weekly_schedule: float[7]          # 每周计划（小时/天，周一至周日）
├── daily_stats: { date_str: seconds } # 每日学习时长统计
├── activity_log: { date_str: count }  # 每日完成视频数
└── videos: Video[]                    # 视频列表

Video（视频）
├── rel_path: string                   # 相对于课程文件夹的路径
├── duration: float                    # 视频时长（秒）
├── watched_duration: float            # 已观看时长（秒）
├── completed: boolean                 # 是否已完成
└── last_watched: ISO datetime | null  # 最后观看时间
```

### 4.2 数据文件示例

```json
{
  "courses": [

      "total_videos": 48,
      "total_duration": 32400.0,
      "start_date": "2026-01-16",

      "daily_stats": {

        {
          "rel_path": "01-基础语法/01-变量与数据类型.mp4",
          "duration": 1200.0,
          "watched_duration": 1200.0,
          "completed": true,
          "last_watched": "2026-01-16T21:00:00"
        }
      ]
    }
  ],
  "activity_log": {
    "2026-01-16": 3,

  },

  }
}
```

## 5. 非功能性需求

### 5.1 性能

| 指标 | 要求 |
|------|------|
| 应用启动时间 | < 2 秒（含数据加载） |
| 视频扫描速度 | 1000 个文件 < 10 秒 |
| 课程列表渲染 | 50 门课程 < 500ms |
| 视频切换延迟 | < 300ms |
| 动画帧率 | ≥ 60fps |
| 内存占用 | 空闲 < 150MB，播放 1080p < 400MB |

### 5.2 可靠性

- JSON 数据文件损坏时，应用不崩溃，提示用户并创建备份
- VLC 检测失败时，自动降级为 Qt Multimedia 引擎
- 视频文件被移动/删除后，列表仍显示但标记为「文件缺失」
- 所有写文件操作使用原子写入（先写临时文件，再替换）

### 5.3 可用性

- 所有操作提供即时视觉反馈
- 删除等破坏性操作需二次确认
- 空状态提供引导提示（如「尚未添加课程，点击 + 开始」）
- 支持仅键盘操作核心功能（空格暂停/播放，方向键调节音量）

### 5.4 可维护性

- 代码遵循 MVC 架构，模型/视图/控制器职责清晰
- 数据访问通过 DataManager 统一接口，视图不直接操作 JSON
- 主题样式通过 ThemeService 统一管理，组件订阅主题变更信号
- 播放器接口抽象（PlayerInterface），便于切换/扩展播放引擎

### 5.5 兼容性

- Windows 10 1809+ / Windows 11
- Python 3.10 ~ 3.12
- PySide6 ≥ 6.5
- VLC ≥ 3.0（可选）
- 显示缩放 100% / 125% / 150% 正确适配

## 6. UI/UX 设计规范

### 6.1 设计语言

- **风格**：现代简约，类 Microsoft Fluent Design
- **圆角**：卡片 12px，按钮 8px/16px（胶囊），窗口 12px
- **阴影**：窗口阴影 blur=20px, offset=(0,2), alpha=80
- **间距**：内容边距 30px，卡片间距 20px，组件内间距 12-15px
- **过渡**：进度条动画 20ms/frame，主题切换 700ms

## 7. 技术架构约束

### 7.1 分层架构

```
┌─────────────────────────────────────────┐
│  Views (UI Layer)                        │
│  main_window / home_view / detail_view  │
│  properties_view / widgets              │
├─────────────────────────────────────────┤
│  Controllers (Logic Layer)               │
│  MainController                          │
├─────────────────────────────────────────┤
│  Services (Business Layer)               │
│  ThemeService / VideoScanner             │
│  PlayerService (VLC / Qt)               │
├─────────────────────────────────────────┤
│  Models (Data Layer)                     │
│  DataManager / Course / Video           │
├─────────────────────────────────────────┤
│  Infrastructure                          │
│  PathManager / Env Setup / JSON I/O     │
└─────────────────────────────────────────┘
```

### 7.2 关键设计原则

1. **单一数据源（Single Source of Truth）**：DataManager 是唯一的数据读写入口
2. **信号驱动（Signal-Driven）**：View 通过 Qt Signal 向 Controller 发送用户意图，Controller 操作 Model 后通知 View 刷新
3. **依赖注入（Dependency Injection）**：Controller 和 Service 在 main.py 中组装，不在 View 内部直接实例化
4. **接口隔离（Interface Segregation）**：播放器抽象为 PlayerInterface，VLC/Qt 各自实现
5. **主题订阅（Theme Subscription）**：所有组件订阅 ThemeService.theme_changed 信号，实现主题实时切换

### 7.3 禁止的反模式

- ❌ View 直接读写 JSON 文件
- ❌ 在 View 中做业务逻辑计算（如余额计算、日期推算）
- ❌ 使用全局变量传递状态
- ❌ 硬编码色值、尺寸、路径
- ❌ 阻塞主线程（文件扫描、视频加载必须异步）

### 7.4 外部依赖

| 包名 | 用途 | 必需? |
|------|------|-------|
| PySide6 | Qt 框架 | ✅ 必需 |
| python-vlc | VLC 播放引擎（含 libvlc.dll） | ❌ 可选 |
| tinytag | 读取视频元数据/时长 | ✅ 必需 |
